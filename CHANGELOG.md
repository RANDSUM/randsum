# Changelog

This monorepo does not maintain a single aggregate changelog. Each publishable
package keeps its own changelog, generated by [changesets](https://github.com/changesets/changesets):

- [`packages/roller/CHANGELOG.md`](packages/roller/CHANGELOG.md) — `@randsum/roller`
- [`packages/games/CHANGELOG.md`](packages/games/CHANGELOG.md) — `@randsum/games`
- [`apps/cli/CHANGELOG.md`](apps/cli/CHANGELOG.md) — `@randsum/cli`

Releases are driven by changesets: contributors add a changeset describing their
change (`bun run changeset`), the CI `version-packages` PR rolls those into version
bumps and per-package changelog entries, and merging that PR publishes to npm.
