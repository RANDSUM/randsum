`@randsum/component-library` ships React components for embedding interactive dice rolling UIs into any web application. Self-contained with bundled styles.

## Install

<Tabs>
  <TabItem label="bun">
    <CodeExample lang="bash" code="bun add @randsum/component-library" />
  </TabItem>
  <TabItem label="npm">
    <CodeExample lang="bash" code="npm install @randsum/component-library" />
  </TabItem>
</Tabs>

Requires `react`, `react-dom`, and `@randsum/roller` as peer dependencies.

## NotationRoller

An interactive dice notation editor. Type a notation string, press roll, and see the total with a step-by-step breakdown panel.

<div class="not-content" style="margin: 1.5rem 0;">
  <NotationRoller client:only="react" defaultNotation="4d6L" />
</div>

<CodeExample code={`import { NotationRoller } from '@randsum/component-library'

export default function App() {
  return <NotationRoller defaultNotation="4d6L" />
}`} />

### Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `defaultNotation` | `string` | `'4d6L'` | Initial dice notation shown in the input (uncontrolled) |
| `notation` | `string` | — | Controlled notation value |
| `className` | `string` | — | Additional class name appended to the root element |

### Theming

The component respects `prefers-color-scheme` automatically. Force a specific theme by setting `data-theme` on any ancestor:

<CodeExample code={`<div data-theme="light">
  <NotationRoller />
</div>

<div data-theme="dark">
  <NotationRoller />
</div>`} />

### Astro usage

In an Astro project, use the `client:only="react"` directive:

<CodeExample code={`<NotationRoller client:only="react" defaultNotation="2d6+3" />`} />

## ErrorBoundary

A React error boundary that catches rendering errors in child components and displays a fallback.

<CodeExample code={`import { ErrorBoundary } from '@randsum/component-library'

<ErrorBoundary>
  <NotationRoller />
</ErrorBoundary>`} />
