RANDSUM is organized as a monorepo with a core dice engine and specialized game packages. Every package is TypeScript-first, ships as ESM + CJS with full type declarations, and is published to npm under `@randsum`.

## Architecture

```
@randsum/roller (core engine)
  |
  +-- @randsum/games (unified package with subpath exports)
       +-- /fifth        (D&D 5e)
       +-- /blades       (Blades in the Dark)
       +-- /pbta          (Powered by the Apocalypse)
       +-- /daggerheart  (Daggerheart)
       +-- /root-rpg     (Root RPG)
       +-- /salvageunion (Salvage Union)
```

All game packages depend only on `@randsum/roller` -- they never depend on each other.

## Design principles

- **Zero core dependencies** -- the roller has no runtime dependencies
- **Throws on invalid input** -- hardcoded notation never throws in practice; validate user-provided strings before rolling
- **Game packages are code-generated** -- they are generated from `.randsum.json` specs and call the core roller directly
- **Strict TypeScript** -- `isolatedDeclarations`, `exactOptionalPropertyTypes`, `noUncheckedIndexedAccess`
- **Bundle size enforced** -- roller is under 10KB, game packages under 8KB
