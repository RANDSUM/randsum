`@randsum/roller` is the core dice rolling engine that powers the entire RANDSUM ecosystem. It provides advanced dice notation support for simple and complex rolling mechanics, with full TypeScript types and zero runtime dependencies.

## Installation

<Tabs>
  <TabItem label="bun">
    <LiveRepl readonly lang="bash" code={`bun add @randsum/roller`} />
  </TabItem>
  <TabItem label="npm">
    <LiveRepl readonly lang="bash" code={`npm install @randsum/roller`} />
  </TabItem>
</Tabs>

## Features

- Advanced dice notation (`4d6L`, `2d20H`, etc.)
- Reroll modifiers
- Exploding dice (standard, compounding, penetrating)
- Drop/keep modifiers
- Cap, replace, and unique modifiers
- Count successes (dice pool systems)
- Pre-arithmetic and total multipliers
- Full TypeScript support with branded types
- Zero runtime dependencies

## Basic usage

<LiveRepl code={`import { roll } from '@randsum/roller'

// Simple d20 roll
roll(20) // Returns 1-20
// or
roll('1d20') // Returns 1-20`} />

<LiveRepl code={`import { roll } from '@randsum/roller'

// Roll 4d6, drop lowest (D&D ability scores)
roll('4d6L')`} />

<LiveRepl code={`import { roll } from '@randsum/roller'

// Roll with advantage (2d20, keep highest)
roll('2d20H')

// Roll with disadvantage (2d20, keep lowest)
roll('2d20L')`} />

<LiveRepl code={`import { roll } from '@randsum/roller'

// Roll 4d6, drop lowest, reroll below 3, exploding
roll('4d6L!R{<3}')`} />

## Argument types

The `roll()` function accepts several input types:

<LiveRepl code={`import { roll } from '@randsum/roller'

// Number: roll 1 die with N sides
roll(20)

// Notation string: parse dice notation
roll('4d6L')

// Options object: full control
roll({
  sides: 6,
  quantity: 4,
  modifiers: { drop: { lowest: 1 } }
})

// Multiple arguments: combined total
roll('1d20', '2d6', '+5')`} />

## Dice notation

Roller supports a comprehensive notation syntax:

| Pattern | Description | Example |
|---|---|---|
| `NdS` | Roll N dice with S sides | `4d6` |
| `+X` / `-X` | Add/subtract from total | `1d20+5` |
| `L` / `H` | Drop lowest/highest | `4d6L` |
| `K` / `kl` | Keep highest/lowest | `4d6K3` |
| R&#123;...&#125; | Reroll conditions | `4d6R...` |
| `!` | Exploding dice | `3d6!` |
| `!!` | Compounding exploding | `3d6!!` |
| `!p` | Penetrating exploding | `3d6!p` |
| `U` | Unique results | `4d20U` |
| C&#123;...&#125; | Cap values | `4d20C...` |
| V&#123;...&#125; | Replace values | `4d20V...` |
| S&#123;...&#125; | Count successes | `5d10S...` |
| `*N` | Pre-arithmetic multiply | `2d6*2` |
| `**N` | Total multiply | `2d6+3**2` |

For the full notation reference, see [Dice Notation](/reference/dice-notation/).

## Result object

<LiveRepl code={`import { roll } from '@randsum/roller'

const result = roll('4d6L+2')

console.log(result.total)  // Final total after all modifiers
console.log(result.rolls)  // Array of RollRecord objects`} />

:::note
`roll()` never throws. If input is invalid, an error is returned in the result: check `result.error` before using `result.total`.
:::

## Validation utilities

<LiveRepl code={`import { isDiceNotation, validateNotation, roll } from '@randsum/roller'

// Type guard
if (isDiceNotation('4d6L')) {
  roll('4d6L') // typed as DiceNotation
}

// Full validation
const validation = validateNotation('4d6L')
console.log(validation)`} />

## Custom random function

Override `Math.random` for reproducible results or cryptographic randomness:

<LiveRepl code={`import { roll } from '@randsum/roller'

const cryptoRandom = () =>
  crypto.getRandomValues(new Uint32Array(1))[0] / 2 ** 32

roll('2d6', { randomFn: cryptoRandom })`} />
