RANDSUM throws dice, not exceptions — but `roll()` will throw if you give it invalid input. The distinction matters for how you structure error handling.

<Aside type="tip">
  Hardcoded notation like `roll('4d6L')` will never throw in practice. Errors only occur when input comes from outside your control: user input, API parameters, configuration files.
</Aside>

## When `roll()` throws

`roll()` throws a `RandsumError` when:

- The notation string is syntactically invalid (`'4dX'`, `'not dice'`)
- The options object has invalid values (`sides: 0`, `quantity: -1`)
- A modifier receives an out-of-range argument

Valid-but-unusual inputs (like `roll(1)` for a single-sided die) are not errors.

## Handling user-provided input

For user-facing inputs — form fields, chat commands, API parameters — validate before rolling. This avoids try/catch and gives you structured error information.

### `validateNotation` — parsed result

<CodeExample code={`import { validateNotation, roll } from '@randsum/roller'

function rollUserInput(input: string) {
  const validation = validateNotation(input)

  if (!validation.valid) {
    // validation.error.message describes what's wrong
    return { error: validation.error.message }
  }

  // Safe to roll — input is valid notation
  const result = roll(input)
  return { total: result.total }
}`} />

<RollerPlayground defaultNotation="4d6L" client:only="react" />

### `isDiceNotation` — type guard

<CodeExample code={`import { isDiceNotation, roll } from '@randsum/roller'

function rollUserInput(input: string) {
  if (!isDiceNotation(input)) {
    return { error: 'Not valid dice notation' }
  }

  // input is now typed as DiceNotation
  const result = roll(input)
  return { total: result.total }
}

console.log(rollUserInput('4d6L'))
console.log(rollUserInput('not-dice'))`} />

## try/catch when you need it

For cases where you need to catch errors from `roll()` directly:

<CodeExample code={`import { roll, RandsumError } from '@randsum/roller'

try {
  const result = roll(userProvidedNotation)
  console.log(result.total)
} catch (e) {
  if (e instanceof RandsumError) {
    console.error(e.message) // Structured RANDSUM error
  } else {
    throw e // Re-throw unexpected errors
  }
}`} />

## Production checklist

- **Hardcoded notation** — no error handling needed; `roll('2d6')` will never throw
- **User-provided notation** — validate with `validateNotation()` or `isDiceNotation()` before calling `roll()`
- **Dynamic options objects** — validate inputs before constructing options; or wrap in try/catch
- **Game packages** — their `roll()` functions also throw on invalid input; validate inputs in the `validate` hook
