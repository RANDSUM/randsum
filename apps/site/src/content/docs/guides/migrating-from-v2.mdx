This guide covers the breaking changes introduced in RANDSUM 3.0 and how to update your code.

## Error handling: result.error to try/catch

The biggest change in 3.0 is how errors are handled. In 2.x, `roll()` never threw — errors were returned in the result object. In 3.x, `roll()` throws a `RandsumError` on invalid input.

<Aside type="danger">
  Code checking `if (result.error)` will silently pass on 3.0 since the field no longer exists. This is the most common migration pitfall.
</Aside>

**Before (2.x):**

<CodeExample code={`import { roll } from '@randsum/roller'

const result = roll(userInput)

if (result.error) {
  console.error(result.error.message)
  return
}

console.log(result.total)`} />

**After (3.x):**

<CodeExample code={`import { roll, RandsumError } from '@randsum/roller'

try {
  const result = roll(userInput)
  console.log(result.total)
} catch (e) {
  if (e instanceof RandsumError) {
    console.error(e.message)
  }
}`} />

For hardcoded notation like `roll('4d6L')`, no error handling is needed — these will never throw in practice. The try/catch is only necessary when input comes from outside your control (user input, API parameters, config files).

You can also validate before rolling to avoid try/catch entirely:

<CodeExample code={`import { validateNotation, roll } from '@randsum/roller'

const validation = validateNotation(userInput)

if (!validation.valid) {
  console.error(validation.error.message)
  return
}

const result = roll(userInput)
console.log(result.total)`} />

## analyze() removed

The `analyze()` function has been removed from the public API. If you were using it for probability analysis, this functionality is no longer exported.

**Before (2.x):**

```typescript
import { analyze } from '@randsum/roller'

const stats = analyze('4d6L')
```

**After (3.x):** Remove `analyze` imports. There is no direct replacement in the public API.

## @randsum/notation extracted

Notation parsing, validation, and types have been extracted into a standalone `@randsum/notation` package. `@randsum/roller` re-exports everything from `@randsum/notation`, so **existing import paths continue to work** without changes.

If you only need parsing and validation (no rolling), you can depend on `@randsum/notation` directly for a smaller bundle:

```typescript
// Both work — choose based on your needs
import { isDiceNotation, validateNotation } from '@randsum/roller'
import { isDiceNotation, validateNotation } from '@randsum/notation'
```

## Game package peer dependency

All game packages now require `@randsum/roller` ^3.0.0 as a peer dependency. Update your roller version before upgrading game packages:

```bash
bun add @randsum/roller@^3.0.0
```

## Migration checklist

1. Search your codebase for `result.error` — replace with try/catch or pre-validation
2. Remove any `analyze()` imports
3. Update `@randsum/roller` to ^3.0.0
4. Update game packages to their latest versions
5. Run your test suite to catch any remaining issues
