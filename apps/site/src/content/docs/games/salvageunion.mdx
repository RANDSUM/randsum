The `@randsum/salvageunion` package provides mechanics for [Salvage Union](https://leyline.press/pages/salvage-union), a post-apocalyptic mech-based tabletop RPG that uses table-based d20 mechanics.

## How Salvage Union dice work

Salvage Union uses a **single d20** roll against a fixed outcome table. Every roll maps to one of five labeled results: **Nailed It** (20), **Success** (11â€“19), **Tough Choice** (6â€“10), **Failure** (2â€“5), and **Cascade Failure** (1). There are no modifiers â€” the d20 roll is read directly against the table. Different in-game situations use different named tables (Core Mechanic, Morale, and others), each with the same five tiers but different narrative descriptions. The `@randsum/salvageunion` package rolls a d20, performs the table lookup, and returns the hit type, table label, narrative description, and raw roll value.

## Overview

- Rolls 1d20 against reference tables
- Results are looked up from game tables with labels and descriptions
- Supports multiple tables (Core Mechanic, Morale, and more)
- Returns structured result data including hit type, label, and description

## API reference

### `roll(tableName?: SalvageUnionTableName)`

**Input:**

| Parameter | Type | Description |
|---|---|---|
| `tableName` | `SalvageUnionTableName` | Table to roll on (defaults to `'Core Mechanic'`) |

**Returns:** `GameRollResult` with:

| Property | Type | Description |
|---|---|---|
| `result` | `SalvageUnionRollRecord` | Table result with metadata |
| `total` | `number` | D20 roll result (1-20) |
| `rolls` | `RollRecord[]` | Raw dice data from the core roller |

### Result structure

The `SalvageUnionRollRecord` includes:

| Property | Type | Description |
|---|---|---|
| `key` | `string` | Internal result identifier |
| `label` | `string` | Human-readable result label |
| `description` | `string` | Detailed result description |
| `table` | `object` | Full table data |
| `tableName` | `string` | Name of the table rolled |
| `roll` | `number` | D20 roll result (1-20) |

## Core Mechanic table

The default Core Mechanic table uses these result tiers:

| Roll | Result |
|---|---|
| 20 | Nailed It -- exceptional success |
| 11-19 | Success -- goal achieved without compromise |
| 6-10 | Tough Choice -- success with complications |
| 2-5 | Failure |
| 1 | Cascade Failure -- critical failure |

:::note
Other tables in Salvage Union have different result tiers and descriptions. The package includes all official tables from the Salvage Union reference.
:::
