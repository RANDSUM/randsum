The `@randsum/blades` package provides mechanics for [Blades in the Dark](https://www.evilhat.com/home/blades-in-the-dark/), a tabletop RPG about a crew of daring scoundrels seeking their fortunes on the haunted streets of an industrial-fantasy city.

## How Blades in the Dark dice work

Blades in the Dark uses a **dice pool** system built around six-sided dice. You assemble a pool of dice based on your action rating plus any bonus dice (from pushing yourself, assistance, or devil's bargains), roll them all, and read only the single highest result: a 6 is a full success, 4–5 is a partial success with a consequence, and 1–3 is a failure. Rolling zero dice (when your effective pool is 0) means rolling two dice and keeping the *lowest*. A **critical** result — two or more 6s — means an exceptional outcome. The `@randsum/blades` package handles all pool assembly, the two-dice-lowest rule for zero-dice rolls, and the outcome tier logic.

## Overview

- Rolls a dice pool of d6s (0-10 dice)
- 0 dice pool: rolls 2d6 and keeps the lowest
- Determines outcome from the highest die in the pool
- Returns one of four outcomes: critical, success, partial, or failure

## API reference

### `roll(count: number)`

**Input:**

| Parameter | Type | Description |
|---|---|---|
| `count` | `number` | Number of dice in the pool (0-10) |

**Returns:** `GameRollResult` with:

| Property | Type | Description |
|---|---|---|
| `result` | `BladesResult` | `'critical' \| 'success' \| 'partial' \| 'failure'` |
| `total` | `number` | Sum of kept dice after modifiers |
| `rolls` | `RollRecord[]` | Raw dice data from the core roller |

## Result interpretation

| Outcome | Condition | Description |
|---|---|---|
| `critical` | Multiple 6s | Critical success -- exceptional outcome |
| `success` | Highest die is 6 | Full success -- you do it |
| `partial` | Highest die is 4-5 | Partial success -- you do it, but with a complication |
| `failure` | Highest die is 1-3 | Bad outcome -- things go wrong |

### Zero dice pool

When the dice pool is 0 (desperate position), the system rolls 2d6 and drops the highest, keeping only the lowest die. This makes failure much more likely.
