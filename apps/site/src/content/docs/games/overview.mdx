## How They Work

Every game package is code-generated from a `.randsum.json` spec and calls `roll()` from `@randsum/roller` directly:

1. **Validate** -- Check the input arguments (e.g., modifier range, dice pool size)
2. **Roll** -- Execute the roll using the core roller engine
3. **Interpret** -- Transform the raw result into a game-specific outcome

<CodeExample code={`import { roll } from '@randsum/games/fifth'

const result = roll({ modifier: 5, rollingWith: 'Advantage' })
// result.result is the game-specific outcome (number for D&D 5e)
// result.total is the final total
// result.rolls contains the raw dice data`} />

## Common patterns

All game packages share these characteristics:

- Export a single `roll()` function as the main API
- Accept game-specific input types (not raw notation strings)
- Return a `GameRollResult` with both raw and interpreted results
- Validate input and throw `SchemaError` on invalid values (importable from each subpath)
- Depend only on `@randsum/roller` (never on each other)
- Ship under 8KB bundled

## Installation

All games are available via the `@randsum/games` package with subpath exports. Install once, import the game you need.

<CodeExample lang="bash" code={`bun add @randsum/games`} />

## Quick comparison

| Package | Dice | Input | Outcomes |
|---|---|---|---|
| `@randsum/games/fifth` | 1d20 or 2d20 | modifier + rollingWith | Numeric total |
| `@randsum/games/blades` | Nd6 pool | count (number) | critical, success, partial, failure |
| `@randsum/games/pbta` | 2d6 | stat + forward + ongoing + rollingWith | strong_hit, weak_hit, miss |
| `@randsum/games/daggerheart` | 2d12 + d6 | modifier + rollingWith + amplifyHope/amplifyFear | Hope/Fear outcome + total |
| `@randsum/games/root-rpg` | 2d6 | bonus (number) | Strong Hit, Weak Hit, Miss |
| `@randsum/games/salvageunion` | 1d20 | tableName (string) | Table result with label + description |
