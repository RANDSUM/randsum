RANDSUM didn't emerge from nothing. It stands on the shoulders of dice rollers, virtual tabletops, and a couple of books that changed how we think about programming.

## Dice Rollers

| Project | What we learned |
|---|---|
| [Sophie's Dice](https://sophiehoulden.com/dice/) | The gold standard for browser-based rolling — RANDSUM's single, expressive `roll()` function owes a direct debt to Sophie's vision that dice rolling should just *work* |

## Virtual Tabletops

| Project | What we learned |
|---|---|
| [Roll20](https://roll20.net/) | The `/roll` command syntax that became a lingua franca — directly informed [Randsum Dice Notation](/notation/randsum-dice-notation/) |
| [Foundry VTT](https://foundryvtt.com/) | Advanced dice modifiers, exploding dice, and an extensible roll API |
| [Owlbear Rodeo](https://www.owlbear.rodeo/) | Proof that simplicity and power aren't mutually exclusive |
| [Alchemy RPG](https://www.alchemyrpg.com/) | Rich game-system integrations |
| [Shard Tabletop](https://www.shardtabletop.com/) | Clean mechanical abstractions |

## Books

| Work | What we learned |
|---|---|
| [_why's (Poignant) Guide to Ruby](https://poignant.guide/) | Code can be *funny*, documentation can be *art* — developer tools should have personality without sacrificing rigor |
| [The Codeless Code](http://thecodelesscode.com/) | Simplicity, restraint, and knowing when *not* to build something — zero dependencies, no runtime network calls, no config files to forget about |
