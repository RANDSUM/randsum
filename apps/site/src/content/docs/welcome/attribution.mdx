RANDSUM didn't emerge from nothing. It stands on the shoulders of dice rollers, virtual tabletops, and a couple of books that changed how we think about programming.

## Sophie's Dice

[Sophie's Dice](https://sophiehoulden.com/dice/) is the gold standard for browser-based dice rolling. Its clean interface, expressive notation support, and sheer joy of use set the bar for what a dice tool should feel like. If you've ever rolled dice on the internet, there's a good chance Sophie's work shaped the experience. RANDSUM's commitment to a single, expressive `roll()` function owes a direct debt to Sophie's vision of dice rolling as something that should just *work*.

## Roll20

[Roll20](https://roll20.net/) brought tabletop RPGs online for millions of players. Its dice notation system — the `/roll` command syntax that became a lingua franca for virtual tabletops — directly informed [Randsum Dice Notation](/notation/randsum-dice-notation/). The idea that you should be able to express complex dice mechanics in a single string traces back to Roll20's pioneering work.

## Virtual Tabletops

The broader VTT ecosystem pushed dice notation forward in ways no single project could:

- [Foundry VTT](https://foundryvtt.com/) — advanced dice modifiers, exploding dice, and an extensible roll API
- [Owlbear Rodeo](https://www.owlbear.rodeo/) — proof that simplicity and power aren't mutually exclusive
- [Alchemy RPG](https://www.alchemyrpg.com/) — rich game-system integrations
- [Shard Tabletop](https://www.shardtabletop.com/) — clean mechanical abstractions

Each of these projects solved problems we learned from. RANDSUM's modifier system, game package architecture, and notation spec all carry fingerprints from the VTT community's collective work.

## _why's (Poignant) Guide to Ruby

[_why's (Poignant) Guide to Ruby](https://poignant.guide/) taught a generation of programmers that code could be *funny*, that documentation could be *art*, and that the best way to learn something is to fall in love with it first. RANDSUM's belief that developer tools should have personality — that a dice library can be opinionated and playful without sacrificing rigor — comes from _why. We may not have cartoon foxes, but the spirit is there.

## The Codeless Code

[The Codeless Code](http://thecodelesscode.com/) is a collection of fables about software engineering, told as koans from a monastery of programmers. Its parables about simplicity, restraint, and knowing when *not* to build something shaped RANDSUM's design philosophy: zero dependencies in the core, no runtime network calls, no configuration files to forget about. The best code is often the code you didn't write.
