RANDSUM uses an extended version of standard dice notation. All notation is **case-insensitive** (`2d8` = `2D8`).

## Basic syntax

The core format is `NdS` where **N** is the number of dice and **S** is the number of sides.

<RollerPlayground defaultNotation="4d6" client:only="react" />
<RollerPlayground defaultNotation="2d12" client:only="react" />

## Zero-bias dice

Roll dice with faces starting at 0 instead of 1. `zN` produces faces 0 to N-1.

| Notation | Description |
|---|---|
| `z6` | One die with faces 0-5 |
| `4z6` | Four dice with faces 0-5 |
| `z10` | One die with faces 0-9 |

<RollerPlayground defaultNotation="1z6" client:only="react" />
<RollerPlayground defaultNotation="4z6" client:only="react" />

## Custom dice faces

Define dice with specific face values using `d{...}`. Supports numeric faces, string faces, or a mix (mixed faces are treated as strings).

| Notation | Description |
|---|---|
| `d{2,3,5,7}` | One die with faces 2, 3, 5, 7 |
| `d{fire,ice,lightning}` | One die with string faces |
| `3d{2,3,5,7}` | Three dice with faces 2, 3, 5, 7 |

Duplicates are allowed for weighting (e.g., `d{1,1,1,2,3}` makes 1 three times as likely). Negatives and zero are valid numeric faces.

<RollerPlayground defaultNotation="1d{2,3,5,7}" client:only="react" />
<RollerPlayground defaultNotation="1d{fire,ice,lightning}" client:only="react" />

## Arithmetic

Add or subtract a fixed value from the total.

| Notation | Description |
|---|---|
| `+N` | Add N to total |
| `-N` | Subtract N from total |

<RollerPlayground defaultNotation="4d6-1" client:only="react" />

## Drop

Remove dice from the result pool.

| Notation | Description |
|---|---|
| `L` | Drop lowest 1 |
| `L2` | Drop lowest 2 |
| `H` | Drop highest 1 |
| `H2` | Drop highest 2 |
| `D{>N}` | Drop rolls over N |
| `D{>=N}` | Drop rolls at or over N |
| `D{<N}` | Drop rolls under N |
| `D{<=N}` | Drop rolls at or under N |
| `D{X,Y}` | Drop exact values X and Y |

<RollerPlayground defaultNotation="4d6L" client:only="react" />
<RollerPlayground defaultNotation="4d6H2" client:only="react" />
<RollerPlayground defaultNotation="4d20D{>17}" client:only="react" />
<RollerPlayground defaultNotation="4d20D{<5}" client:only="react" />
<RollerPlayground defaultNotation="4d20D{8,12}" client:only="react" />

## Keep

Keep specific dice from the result (complement of drop).

| Notation | Description |
|---|---|
| `K` | Keep highest 1 |
| `K3` | Keep highest 3 |
| `kl` | Keep lowest 1 |
| `kl2` | Keep lowest 2 |
| `KM` | Keep middle (drop 1 lowest and 1 highest) |
| `KM2` | Keep middle, dropping N from each end |

<RollerPlayground defaultNotation="4d6K3" client:only="react" />
<RollerPlayground defaultNotation="4d6kl2" client:only="react" />
<RollerPlayground defaultNotation="2d20K" client:only="react" />
<RollerPlayground defaultNotation="2d20kl" client:only="react" />
<RollerPlayground defaultNotation="6d6KM" client:only="react" />

:::note
Keeping N highest is equivalent to dropping (quantity - N) lowest. For example, `4d6K3` is the same as `4d6L`.
:::

:::note
`KM` (keep middle) is sugar for dropping N from the lowest and N from the highest. `6d6KM` drops 1 from each end, keeping the middle 4 dice. `6d6KM2` drops 2 from each end, keeping the middle 2 dice.
:::

## Cap

Limit roll values to a specific range.

| Notation | Description |
|---|---|
| `C{>N}` | Cap rolls over N to N |
| `C{<N}` | Cap rolls under N to N |
| `C{<N,>M}` | Cap both ends |

<RollerPlayground defaultNotation="4d20C{>18}" client:only="react" />
<RollerPlayground defaultNotation="4d20C{<3}" client:only="react" />
<RollerPlayground defaultNotation="4d20C{<2,>19}" client:only="react" />

## Reroll

Reroll dice matching certain conditions.

| Notation | Description |
|---|---|
| `R{>N}` | Reroll results over N |
| `R{>=N}` | Reroll results at or over N |
| `R{<N}` | Reroll results under N |
| `R{<=N}` | Reroll results at or under N |
| `R{X,Y}` | Reroll exact values X, Y |
| `R{<N}M` | Reroll under N, max M attempts |

<RollerPlayground defaultNotation="4d20R{>17}" client:only="react" />
<RollerPlayground defaultNotation="4d20R{<5}" client:only="react" />
<RollerPlayground defaultNotation="4d20R{8,12}" client:only="react" />
<RollerPlayground defaultNotation="4d20R{<5}3" client:only="react" />

### Reroll once

`ro{...}` is sugar for `R{...}1` -- reroll matching dice at most once. Supports all comparison operators. Case-insensitive.

| Notation | Description |
|---|---|
| `ro{<N}` | Reroll under N once |
| `ro{>N}` | Reroll over N once |
| `ro{X,Y}` | Reroll exact values once |

<RollerPlayground defaultNotation="4d20ro{<5}" client:only="react" />

## Replace

Replace specific results with new values.

| Notation | Description |
|---|---|
| `V{X=Y}` | Replace Xs with Y |
| `V{>N=Y}` | Replace results over N with Y |
| `V{<N=Y}` | Replace results under N with Y |

<RollerPlayground defaultNotation="4d20V{8=12}" client:only="react" />
<RollerPlayground defaultNotation="4d20V{>17=20}" client:only="react" />
<RollerPlayground defaultNotation="4d20V{<5=1}" client:only="react" />

## Unique

Force all dice in a pool to show different values.

| Notation | Description |
|---|---|
| `U` | All results must be unique |
| `U{X,Y}` | Unique except X and Y can repeat |

<RollerPlayground defaultNotation="4d20U" client:only="react" />
<RollerPlayground defaultNotation="4d20U{5,10}" client:only="react" />

## Explode

Roll additional dice when a die shows its maximum value.

| Notation | Description |
|---|---|
| `!` | Explode: one additional die per max result (single pass) |

<RollerPlayground defaultNotation="4d20!" client:only="react" />

When a die shows its maximum value, it "explodes" — one new die is rolled and added to the result. This is a single pass: the newly added dice are not checked for further explosions. For recursive explosions, use compound (`!!`) or penetrate (`!p`).

**Example:** `3d6!` rolls `[6, 4, 6]`. The two 6s explode, adding `[5, 3]`. Final result: `[6, 4, 6, 5, 3]` = 24.

## Compound

Exploding dice that add to the triggering die instead of creating new dice.

| Notation | Description |
|---|---|
| `!!` | Compound once per die |
| `!!N` | Compound with max depth N |
| `!!0` | Compound unlimited (capped at 1000) |

<RollerPlayground defaultNotation="3d6!!" client:only="react" />
<RollerPlayground defaultNotation="3d6!!5" client:only="react" />

**Example:** `1d6!!` rolls 6. This compounds, rolling 4. The die value becomes 6 + 4 = 10. Unlike regular exploding, this does not create new dice -- it modifies the existing die.

**Difference from explode:**
- **Explode (`!`)**: Creates new dice -- `[6, 4, 6]` becomes `[6, 4, 6, 5, 3]` (5 dice)
- **Compound (`!!`)**: Modifies existing die -- `[6, 4, 6]` becomes `[15, 4, 12]` (still 3 dice)

## Penetrate

Exploding dice where each subsequent explosion subtracts 1 (Hackmaster-style).

| Notation | Description |
|---|---|
| `!p` | Penetrate once per die |
| `!pN` | Penetrate with max depth N |
| `!p0` | Penetrate unlimited (capped at 1000) |

<RollerPlayground defaultNotation="3d6!p" client:only="react" />
<RollerPlayground defaultNotation="3d6!p5" client:only="react" />

**Example:** `1d6!p` rolls 6. This penetrates, rolling 5. The value added is 5 - 1 = 4, so the die becomes 6 + 4 = 10.

## Pre-Arithmetic Multiply

Multiply the dice sum before adding/subtracting arithmetic modifiers.

| Notation | Description |
|---|---|
| `*N` | Multiply dice sum by N |

<RollerPlayground defaultNotation="2d6*2+3" client:only="react" />
<RollerPlayground defaultNotation="4d6*3" client:only="react" />

**Example:** `2d6*2+3` rolls `[4, 5]` = 9. Multiplied by 2 = 18. Plus 3 = 21.

## Margin of success

`ms{N}` is sugar for `-N`. Useful for readability when expressing target-number checks.

| Notation | Description |
|---|---|
| `ms{N}` | Subtract N from total (margin of success) |

<RollerPlayground defaultNotation="1d20ms{15}" client:only="react" />

**Example:** `1d20ms{15}` is equivalent to `1d20-15`. Roll 14 = margin of -1. Roll 18 = margin of +3.

## Sorting

Sort the displayed dice results. Sorting is display-only -- it does not affect the total.

| Notation | Description |
|---|---|
| `s` | Sort ascending (default) |
| `sa` | Sort ascending (explicit) |
| `sd` | Sort descending |

<RollerPlayground defaultNotation="6d6s" client:only="react" />
<RollerPlayground defaultNotation="6d6sd" client:only="react" />

:::note
Sorting does not conflict with count successes (`S{7}`). The lowercase `s`/`sa`/`sd` tokens are distinct from the uppercase `S{...}` syntax.
:::

## Count successes

Count dice meeting a threshold instead of summing values. Used in dice pool systems.

| Notation | Description |
|---|---|
| `S{N}` | Count dice >= N |
| `S{N,B}` | Count successes >= N, subtract botches &lt;= B |

<RollerPlayground defaultNotation="5d10S{7}" client:only="react" />
<RollerPlayground defaultNotation="5d10S{7,1}" client:only="react" />

**Example:** `5d10S{7}` rolls `[8, 3, 10, 6, 9]`. Successes >= 7: `[8, 10, 9]` = 3 successes.

## Total Multiply

Multiply the entire final total after all other modifiers.

| Notation | Description |
|---|---|
| `**N` | Multiply final total by N |

<RollerPlayground defaultNotation="2d6+3**2" client:only="react" />
<RollerPlayground defaultNotation="4d6L+2**3" client:only="react" />

**Difference from pre-arithmetic multiplier:**
- **Pre-arithmetic (`*`)**: `2d6*2+3` = (sum x 2) + 3
- **Total (`**`)**: `2d6+3**2` = (sum + 3) x 2

## Combining modifiers

Modifiers can be chained together. They are applied in a fixed priority order:

| Priority | Modifier | Notation | Description |
|---|---|---|---|
| 10 | Cap | `C{...}` | Limit roll values to a range |
| 20 | Drop | `H`, `L` | Remove dice from pool |
| 21 | Keep | `K`, `kl`, `KM` | Keep dice in pool |
| 30 | Replace | `V{...}` | Replace specific values |
| 40 | Reroll | `R{...}`, `ro{...}` | Reroll dice matching conditions |
| 50 | Explode | `!` | Roll additional dice on max |
| 51 | Compound | `!!` | Add explosion to existing die |
| 52 | Penetrate | `!p` | Add explosion minus 1 to die |
| 60 | Unique | `U` | Ensure no duplicate values |
| 85 | Multiply | `*N` | Multiply dice sum (pre-arithmetic) |
| 90 | Plus | `+N` | Add to total |
| 91 | Minus | `-N`, `ms{N}` | Subtract from total |
| 92 | Sort | `s`, `sa`, `sd` | Sort displayed results |
| 95 | Count Successes | `S{...}` | Count dice meeting threshold |
| 100 | Total Multiply | `**N` | Multiply entire final total |

<RollerPlayground defaultNotation="4d6R{<3}L" client:only="react" />
<RollerPlayground defaultNotation="4d6L+2" client:only="react" />
<RollerPlayground defaultNotation="3d6!!*2+3" client:only="react" />
<RollerPlayground defaultNotation="4d6K3!+2" client:only="react" />
