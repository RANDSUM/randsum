RANDSUM uses an extended version of standard dice notation. All notation is **case-insensitive** (`2d8` = `2D8`).

## Basic syntax

The core format is `NdS` where **N** is the number of dice and **S** is the number of sides.

<RollerPlayground defaultNotation="4d6" client:only="react" />
<RollerPlayground defaultNotation="2d12" client:only="react" />

## Custom dice faces

Define a die with explicit face values using curly braces. Faces can be numeric (including negative and zero) or strings.

| Notation | Description |
|---|---|
| `d{2,3,5,7}` | Die with faces 2, 3, 5, 7 |
| `d{-1,0,1}` | Die with negative, zero, and positive faces |
| `d{fire,ice,lightning}` | Die with string faces |
| `3d{1,1,2}` | Three dice with weighted faces (1 appears twice) |

All notation is **case-insensitive**.

<RollerPlayground defaultNotation="d{2,3,5,7}" client:only="react" />
<RollerPlayground defaultNotation="3d{-1,0,1}" client:only="react" />
<RollerPlayground defaultNotation="d{fire,ice,lightning}" client:only="react" />

Custom faces are useful for non-standard dice like direction dice, hit location dice, or any die where the faces are not a simple 1-to-N range. Weighted distributions are achieved by repeating face values (e.g., `d{1,1,2}` has a 2/3 chance of rolling 1).

:::note
String-face dice produce string results. Arithmetic modifiers (`+N`, `-N`, `*N`, `**N`) and numeric comparisons (`C{>N}`, `S{N}`, etc.) are not applicable to string-face dice. Use numeric faces when you need mathematical operations on the results.
:::

## Zero-bias dice

A zero-bias die rolls 0 to S-1 instead of 1 to S. Useful for zero-indexed systems and probability calculations.

| Notation | Description |
|---|---|
| `z6` | Roll 0-5 (zero-bias d6) |
| `z10` | Roll 0-9 (zero-bias d10) |
| `3z10` | Three zero-bias d10s |

All notation is **case-insensitive** (`z6` = `Z6`).

<RollerPlayground defaultNotation="z6" client:only="react" />
<RollerPlayground defaultNotation="3z10" client:only="react" />

:::note
`zN` is sugar for rolling a dN with `minus: 1` applied. `z6` produces the same distribution as `1d6-1`. Zero-bias dice support all standard modifiers.
:::

## Drop

Remove dice from the result pool.

| Notation | Description |
|---|---|
| `L` | Drop lowest 1 |
| `L2` | Drop lowest 2 |
| `H` | Drop highest 1 |
| `H2` | Drop highest 2 |
| `D{>N}` | Drop rolls over N |
| `D{>=N}` | Drop rolls at or over N |
| `D{<N}` | Drop rolls under N |
| `D{<=N}` | Drop rolls at or under N |
| `D{X,Y}` | Drop exact values X and Y |

<RollerPlayground defaultNotation="4d6H2" client:only="react" />
<RollerPlayground defaultNotation="4d20D{>17}" client:only="react" />
<RollerPlayground defaultNotation="4d20D{<5}" client:only="react" />
<RollerPlayground defaultNotation="4d20D{8,12}" client:only="react" />

## Keep

Keep specific dice from the result (complement of drop).

| Notation | Description |
|---|---|
| `K` | Keep highest 1 |
| `K3` | Keep highest 3 |
| `kl` | Keep lowest 1 |
| `kl2` | Keep lowest 2 |

<RollerPlayground defaultNotation="4d6K3" client:only="react" />
<RollerPlayground defaultNotation="4d6kl2" client:only="react" />
<RollerPlayground defaultNotation="2d20K" client:only="react" />
<RollerPlayground defaultNotation="2d20kl" client:only="react" />

:::note
Keeping N highest is equivalent to dropping (quantity - N) lowest. For example, `4d6K3` is the same as `4d6L`.
:::

## Keep middle

Drop dice from both ends of the sorted pool. Sugar for combining drop lowest + drop highest.

| Notation | Description |
|---|---|
| `KM` | Keep middle (drop 1 lowest and 1 highest) |
| `KMN` | Keep middle, dropping N from each end |

All notation is **case-insensitive** (`km` = `KM`).

<RollerPlayground defaultNotation="6d6KM" client:only="react" />
<RollerPlayground defaultNotation="8d6KM2" client:only="react" />

:::note
`KM` is sugar over drop. `6d6KM` is equivalent to `6d6LH` (drop lowest 1 and highest 1). `8d6KM2` drops 2 from each end, keeping the middle 4.
:::

## Cap

Limit roll values to a specific range.

| Notation | Description |
|---|---|
| `C{>N}` | Cap rolls over N to N |
| `C{<N}` | Cap rolls under N to N |
| `C{<N,>M}` | Cap both ends |

<RollerPlayground defaultNotation="4d20C{<3}" client:only="react" />

## Reroll

Reroll dice matching certain conditions.

| Notation | Description |
|---|---|
| `R{>N}` | Reroll results over N |
| `R{>=N}` | Reroll results at or over N |
| `R{<N}` | Reroll results under N |
| `R{<=N}` | Reroll results at or under N |
| `R{X,Y}` | Reroll exact values X, Y |
| `R{<N}M` | Reroll under N, max M attempts |

<RollerPlayground defaultNotation="4d20R{>17}" client:only="react" />
<RollerPlayground defaultNotation="4d20R{<5}" client:only="react" />
<RollerPlayground defaultNotation="4d20R{8,12}" client:only="react" />
<RollerPlayground defaultNotation="4d20R{<5}3" client:only="react" />

## Reroll once

Reroll dice matching a condition, but only attempt one reroll per die. Sugar for `R{...}1` (reroll with max 1 attempt).

| Notation | Description |
|---|---|
| `ro{>N}` | Reroll once if over N |
| `ro{>=N}` | Reroll once if at or over N |
| `ro{<N}` | Reroll once if under N |
| `ro{<=N}` | Reroll once if at or under N |
| `ro{X,Y}` | Reroll once if exact value X or Y |

All notation is **case-insensitive** (`ro` = `RO`).

<RollerPlayground defaultNotation="4d20ro{<5}" client:only="react" />
<RollerPlayground defaultNotation="2d6ro{1}" client:only="react" />

:::note
`ro{<5}` is exactly equivalent to `R{<5}1`. Use `ro` when you want to make the single-attempt intent explicit, which is common in D&D 5e features like the Halfling Lucky trait or Great Weapon Fighting.
:::

## Replace

Replace specific results with new values.

| Notation | Description |
|---|---|
| `V{X=Y}` | Replace Xs with Y |
| `V{>N=Y}` | Replace results over N with Y |
| `V{<N=Y}` | Replace results under N with Y |

<RollerPlayground defaultNotation="4d20V{8=12}" client:only="react" />
<RollerPlayground defaultNotation="4d20V{>17=20}" client:only="react" />

## Explode

Roll additional dice when a die shows its maximum value.

| Notation | Description |
|---|---|
| `!` | Explode: one additional die per max result (single pass) |

<RollerPlayground defaultNotation="4d20!" client:only="react" />

When a die shows its maximum value, it "explodes" — one new die is rolled and added to the result. This is a single pass: the newly added dice are not checked for further explosions. For recursive explosions, use compound (`!!`) or penetrate (`!p`).

**Example:** `3d6!` rolls `[6, 4, 6]`. The two 6s explode, adding `[5, 3]`. Final result: `[6, 4, 6, 5, 3]` = 24.

## Compound

Exploding dice that add to the triggering die instead of creating new dice.

| Notation | Description |
|---|---|
| `!!` | Compound once per die |
| `!!N` | Compound with max depth N |
| `!!0` | Compound unlimited (capped at 1000) |

<RollerPlayground defaultNotation="3d6!!" client:only="react" />
<RollerPlayground defaultNotation="3d6!!5" client:only="react" />

**Example:** `1d6!!` rolls 6. This compounds, rolling 4. The die value becomes 6 + 4 = 10. Unlike regular exploding, this does not create new dice -- it modifies the existing die.

**Difference from explode:**
- **Explode (`!`)**: Creates new dice -- `[6, 4, 6]` becomes `[6, 4, 6, 5, 3]` (5 dice)
- **Compound (`!!`)**: Modifies existing die -- `[6, 4, 6]` becomes `[15, 4, 12]` (still 3 dice)

## Penetrate

Exploding dice where each subsequent explosion subtracts 1 (Hackmaster-style).

| Notation | Description |
|---|---|
| `!p` | Penetrate once per die |
| `!pN` | Penetrate with max depth N |
| `!p0` | Penetrate unlimited (capped at 1000) |

<RollerPlayground defaultNotation="3d6!p" client:only="react" />
<RollerPlayground defaultNotation="3d6!p5" client:only="react" />

**Example:** `1d6!p` rolls 6. This penetrates, rolling 5. The value added is 5 - 1 = 4, so the die becomes 6 + 4 = 10.

## Explode sequence

Explode through a sequence of die sizes instead of re-rolling the same die. When a die shows its maximum, the next die in the sequence is rolled and added. The final die in the sequence repeats indefinitely (safety capped).

| Notation | Description |
|---|---|
| `!s{4,6,8}` | Explode through d4, then d6, then d8 (d8 repeats) |
| `!i` | Inflation: explode UP through the TTRPG standard set |
| `!r` | Reduction: explode DOWN through the TTRPG standard set |

All notation is **case-insensitive** (`!s` = `!S`, `!i` = `!I`, `!r` = `!R`).

<RollerPlayground defaultNotation="1d4!s{4,6,8}" client:only="react" />
<RollerPlayground defaultNotation="1d6!i" client:only="react" />
<RollerPlayground defaultNotation="1d20!r" client:only="react" />

**Example walkthrough:** `1d4!s{4,6,8}` rolls a d4 and gets 4 (max). This triggers the next die in the sequence: roll a d6, getting 6 (max). Next in sequence: roll a d8, getting 5. Final result: 4 + 6 + 5 = 15. If the d8 had also been max (8), it would roll another d8 (the final die repeats).

**TTRPG Standard Die Set:** `[4, 6, 8, 10, 12, 20, 100]`

- **Inflation (`!i`)**: Starting from the current die size, explode upward through the standard set. A d6 with `!i` would sequence through d8, d10, d12, d20, d100, then repeat d100.
- **Reduction (`!r`)**: Starting from the current die size, explode downward through the standard set. A d20 with `!r` would sequence through d12, d10, d8, d6, d4, then repeat d4.

:::note
`!i` and `!r` are sugar for `!s{...}` with the TTRPG standard die set. Non-standard starting die sizes snap to the nearest value in the set in the appropriate direction.
:::

## Pre-Arithmetic Multiply

Multiply the dice sum before adding/subtracting arithmetic modifiers.

| Notation | Description |
|---|---|
| `*N` | Multiply dice sum by N |

<RollerPlayground defaultNotation="2d6*2+3" client:only="react" />
<RollerPlayground defaultNotation="4d6*3" client:only="react" />

**Example:** `2d6*2+3` rolls `[4, 5]` = 9. Multiplied by 2 = 18. Plus 3 = 21.

## Count successes

Count dice meeting a threshold instead of summing values. Used in dice pool systems.

| Notation | Description |
|---|---|
| `S{N}` | Count dice >= N |
| `S{N,B}` | Count successes >= N, subtract botches &lt;= B |

<RollerPlayground defaultNotation="5d10S{7}" client:only="react" />
<RollerPlayground defaultNotation="5d10S{7,1}" client:only="react" />

**Example:** `5d10S{7}` rolls `[8, 3, 10, 6, 9]`. Successes >= 7: `[8, 10, 9]` = 3 successes.

## Margin of success

Calculate how far above or below a target number the roll landed. Sugar for subtracting a fixed value.

| Notation | Description |
|---|---|
| `ms{N}` | Subtract N from the total (margin relative to target N) |

All notation is **case-insensitive** (`ms` = `MS`).

<RollerPlayground defaultNotation="1d20+5ms{15}" client:only="react" />
<RollerPlayground defaultNotation="3d6ms{10}" client:only="react" />

A positive result means the roll exceeded the target; zero means exactly met; negative means the roll fell short. This is commonly used in systems with Difficulty Classes (DCs) or Target Numbers (TNs).

:::note
`ms{N}` is sugar for `-N` (the `minus` modifier). `1d20+5ms{15}` is equivalent to `1d20+5-15`, which simplifies to `1d20-10`. The `ms` form makes the intent clearer when checking against a specific target.
:::

## Total Multiply

Multiply the entire final total after all other modifiers.

| Notation | Description |
|---|---|
| `**N` | Multiply final total by N |

<RollerPlayground defaultNotation="2d6+3**2" client:only="react" />
<RollerPlayground defaultNotation="4d6L+2**3" client:only="react" />

**Difference from pre-arithmetic multiplier:**
- **Pre-arithmetic (`*`)**: `2d6*2+3` = (sum x 2) + 3
- **Total (`**`)**: `2d6+3**2` = (sum + 3) x 2

## Special dice

### Percentile die (`d%`)

A percentile die rolls 1-100. Used in Call of Cthulhu, Warhammer Fantasy, and any system with percentage-based resolution.

| Argument | Equivalent | Description |
|---|---|---|
| `d%` | `roll(100)` | Roll one d100 |

<CodeExample code={`import { roll } from '@randsum/roller'

roll('d%')   // Roll 1d100
roll('D%')   // Case-insensitive
roll(100)    // Equivalent numeric form`} />

`d%` does not support a quantity prefix. To roll multiple percentile dice, pass multiple arguments: `roll('d%', 'd%')`.

### Fate/Fudge dice (`dF`)

Fate dice (also called Fudge dice) produce results of -1, 0, or +1 per die. The standard Fate Core roll is `4dF`, giving a range of -4 to +4. An extended variant (`dF.2`) uses five faces: -2, -1, 0, +1, +2.

| Argument | Faces | Range per die | Description |
|---|---|---|---|
| `dF` | -1, 0, +1 | -1 to +1 | Standard Fate/Fudge die |
| `dF.1` | -1, 0, +1 | -1 to +1 | Explicit standard variant |
| `dF.2` | -2, -1, 0, +1, +2 | -2 to +2 | Extended Fudge die |
| `4dF` | -1, 0, +1 | -4 to +4 | Standard Fate Core roll |
| `4dF.2` | -2, -1, 0, +1, +2 | -8 to +8 | Four extended dice |

<CodeExample code={`import { roll } from '@randsum/roller'

roll('dF')     // One Fate die: -1, 0, or +1
roll('4dF')    // Standard Fate Core roll (4 dice, range -4 to +4)
roll('dF.2')   // Extended Fudge die: -2, -1, 0, +1, or +2
roll('4dF.2')  // Four extended dice (range -8 to +8)`} />

:::note
Neither `d%` nor `dF` support inline notation modifiers -- there is no `d%L` or `4dFkh3`. To apply modifiers to these dice, use the options object form.
:::

## Combining modifiers

Modifiers can be chained together. They are applied in a fixed priority order:

| Priority | Modifier | Notation | Description |
|---|---|---|---|
| 10 | Cap | `C{...}` | Limit roll values to a range |
| 20 | Drop | `H`, `L` | Remove dice from pool |
| 21 | Keep | `K`, `kl` | Keep dice in pool |
| 30 | Replace | `V{...}` | Replace specific values |
| 40 | Reroll | `R{...}` | Reroll dice matching conditions |
| 50 | Explode | `!` | Roll additional dice on max |
| 51 | Compound | `!!` | Add explosion to existing die |
| 52 | Penetrate | `!p` | Add explosion minus 1 to die |
| 53 | Explode Sequence | `!s{...}` | Explode through die size sequence |
| 60 | Unique | `U` | Ensure no duplicate values |
| 85 | Multiply | `*N` | Multiply dice sum (pre-arithmetic) |
| 90 | Plus | `+N` | Add to total |
| 91 | Minus | `-N` | Subtract from total |
| 95 | Count Successes | `S{...}` | Count dice meeting threshold |
| 100 | Total Multiply | `**N` | Multiply entire final total |

<RollerPlayground defaultNotation="4d6L+2" client:only="react" />
<RollerPlayground defaultNotation="3d6!!*2+3" client:only="react" />
<RollerPlayground defaultNotation="4d6K3!+2" client:only="react" />

## Multiple groups
