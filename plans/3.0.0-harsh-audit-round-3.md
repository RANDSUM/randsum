## Round 3 Blockers (Must Fix Before Merging)

### 1. CLI: `catalog:` protocol in published `react` dependency
**File:** `apps/cli/package.json:48`
```json
"react": "catalog:"
```
`catalog:` is Bun-workspace-only. npm does not resolve it. The published package.json literally contains `"react": "catalog:"`. Anyone running `npm install -g @randsum/cli` fails. This is a broken publish waiting to happen.

**Fix:** Replace with `"react": "^19.2.0"` (the resolved value from root catalog).

---

### 2. Component library bundles workspace deps it should externalize
**File:** `packages/component-library/bunup.config.ts:7`
```typescript
external: ['react', 'react-dom', 'react/jsx-runtime', 'react/jsx-dev-runtime'],
```
Only React is externalized. `@randsum/roller`, `@randsum/display-utils`, and `@randsum/notation` are listed as runtime `dependencies` in `package.json:43-46` but absent from `external`. The bundled dist includes full copies of all three packages. Consumers get them again as separate node_modules. Double code, version mismatch risk, tree-shaking broken.

**Fix:** Add `'@randsum/roller'`, `'@randsum/display-utils'`, `'@randsum/notation'` to the `external` array.

---

### 3. Component library CSS injection requires `unsafe-inline` CSP
**File:** `packages/component-library/bunup.config.ts:12-14`
```typescript
css: { inject: true }
```
This was flagged in Round 1 and Round 2 as a "Significant Issue." It remains unfixed and is now a blocker. Injecting CSS as JavaScript requires `style-src 'unsafe-inline'` in any CSP. The component library is unusable in Next.js apps with strict CSP, enterprise deployments, or any app that already bans inline styles. There is no escape hatch. CSS includes hardcoded dark-mode `#0d1117` backgrounds that will clash with consumer themes.

**Fix:** Either (a) extract CSS and document that consumers must import `@randsum/component-library/styles.css`, or (b) document the CSP requirement prominently in the README before publishing. If neither is done, this will cause silent failures in a significant percentage of target environments.

---

### 4. `check:all` diverges from CI — creates false confidence
**File:** `package.json:130`

`check:all` runs: `lint, format:check, typecheck, test, build, size, site:build`
CI also runs: `check:exports`
`check:all` does not.

A contributor follows CONTRIBUTING.md (line 29), runs `bun run check:all`, sees green, pushes, and gets a CI failure on unused exports. This was flagged independently by two auditors.

**Fix:** Add `&& bun run check:exports` to `check:all` before `site:build`.

---

### 5. Lefthook pre-push runs `size` without first running `build`
**File:** `lefthook.yml:23-24`

Pre-push checks size against `dist/index.js` paths. If dist is stale or missing, the size check either errors or checks wrong artifacts. CI runs `build` before `size` (main.yml:44-46). Lefthook does not.

**Fix:** Add a `build` step to lefthook pre-push before `size`.

---

### 6. Site: All 6 game package source URLs are 404s
**File:** `apps/site/src/utils/packageData.ts:45,55,67,78,89,100`
```
sourceUrl: 'https://github.com/RANDSUM/randsum/tree/main/packages/blades'   // 404
sourceUrl: 'https://github.com/RANDSUM/randsum/tree/main/packages/daggerheart' // 404
sourceUrl: 'https://github.com/RANDSUM/randsum/tree/main/packages/fifth'    // 404
sourceUrl: 'https://github.com/RANDSUM/randsum/tree/main/packages/root-rpg' // 404
sourceUrl: 'https://github.com/RANDSUM/randsum/tree/main/packages/salvageunion' // 404
sourceUrl: 'https://github.com/RANDSUM/randsum/tree/main/packages/pbta'     // 404
```
Game packages live in `games/`, not `packages/`. Every "View Source" link on the site for game packages is a dead link.

**Fix:** Change all 6 to `games/<name>`.

---

### 7. Discord bot: `/notation` links to a file that doesn't exist
**File:** `apps/discord-bot/src/commands/notation.ts:97`
```typescript
.setURL('https://github.com/RANDSUM/randsum/blob/main/packages/roller/GUIDE.md')
```
`packages/roller/GUIDE.md` does not exist. The correct file is `packages/roller/RANDSUM_DICE_NOTATION.md`. Every user who clicks the link in Discord gets a 404.

**Fix:** Update URL to point to `RANDSUM_DICE_NOTATION.md`.

---

### 8. Discord bot: `/notation` teaches users three wrong notation syntaxes
**File:** `apps/discord-bot/src/commands/notation.ts:25-41`

Three inline examples use invalid notation:
- Cap: `3d10C5` (shown) → correct: `3d10C{5}` (braces required)
- Reroll: `2d20R1` (shown) → correct: `2d20R{1}` (braces required)
- Replace: `3d6r1:6` (shown) → correct: `3d6V{1=6}` (wrong letter, wrong separator, missing braces)

The bot is actively teaching users notation that the roller will reject. These will cause support confusion immediately after launch.

**Fix:** Correct all three examples against `RANDSUM_DICE_NOTATION.md`.
