# RANDSUM Monorepo Refresh Audit — 2026-06-12

Branch: `jarvis/refresh`. Method: adversarial multi-agent workflow — 11 reviewers across
best-practices / DRY / UX / DX / strategic dimensions, each finding independently
verified by a skeptic agent instructed to refute it (120 agents, ~4.5M tokens).
Raw verified findings: `audit/refresh-2026-06-12-raw-findings.json`.

## 1. Executive summary

RANDSUM is in strong internal health following the recently-shipped 30-item audit (#1056):
the core roller engine is well-tested (1100+ tests, strict TS, clean error hierarchy),
publishing is sorted, and CI gates are wired. **No new P0s — nothing blocks shipping.**

Dominant themes:

1. **Runtime↔codegen duplication** in the games pipeline — the same domain logic
   (operator maps, leaf resolution, degree-of-success, range matching) is implemented twice
   with real silent-drift risk and no test asserting the two agree.
2. **An under-decided dice-ui dual-target strategy** — neither web-first-with-escape-hatch nor
   true dual-mode; ~40% duplication and a live native-only crash (`DocModal` missing from the
   native barrel, masked by a hand-written `.d.ts` + a test mock).
3. **Doc-vs-code drift** — README/llms.txt/CONTRIBUTING misadvertise removed features.

Highest-correctness issue: the **count-modifier collision** — `roll('3d6S{3}F{1}')` silently
drops a condition (three schemas all key on `count`, last-match-wins) and returns wrong
results through the public API.

## 2. Strategic verdict

### DROP (dead weight / over-engineering)

- **`scripts/help.ts` stale entries** — lists non-existent `mcp:compile`/`mcp:compile:all` and
  wrong release script names (`changeset:version`/`changeset:publish` vs real
  `version-packages`/`publish`).
- **`QuickReferenceGrid.native.tsx` (866 LOC)** — violates the project's own ADR-003 (use-dom
  WebView is the documented native default; the `.native.tsx` port was deferred until WebView
  perf became a blocker, which never happened). Delete it and rely on use-dom, or amend ADR-003.
- **Resist these over-engineered "fixes":** auto-patching `metro.config.js`, a full `CodeExpr`
  AST for codegen, badge-generation CI hooks, shared-`scripts.json` machinery. All cost more
  than the trivial problems they solve.

### EXPAND (under-invested)

- **Multi-roll spec support** (P1) — schema/codegen allow exactly one `roll` per spec, blocking
  Blades resistance rolls and D&D 5e ability-score generation/saves. The IR and emit loop already
  iterate named rolls; only `getRollDefinitions()` (typeGuards.ts:42) and the singular `roll`
  schema property block it. Cheap unblock for two flagship games.
- **Non-standard die types** (P2) — `PoolDefinition.sides: IntegerOrInput` can't express
  Fate/Fudge or custom faces, blocking the planned Fate Core package. Roller already supports
  these; add a `faces: number[]` / `notation` escape hatch.
- **Dual-consumer validation test** (P1) — no test runs both generated `roll()` and the runtime
  pipeline on the same input and asserts agreement; drift can accumulate silently.
- **Expo test coverage** — CI runs 1 of 11 expo test files; the full suite has ~42
  failing/erroring tests hiding behind the gate.

### REFINE (works but rough)

- **Count-modifier collision** (P1) — normalize sugar (`S{N}`/`F{N}`) to `#{}` pre-parse, or
  consolidate into one count schema that validates + rejects conflicts.
- **Runtime↔codegen duplication** — extract shared declarative descriptors both the runtime
  predicate and the string emitter consume.
- **`sideEffects: true` regression** — ADR-018 says the value should be `["./src/**/*"]`. The
  audit flagged this as a one-line fix, but it is **NOT safe**: applying it breaks the
  `@randsum/cli` bundle at runtime (`Error: a2 is not defined` — bunup 0.16.32 still DCE's a
  binding the cli needs once roller's dist is marked side-effect-free). Left at `true`; ADR-018
  is effectively regressed and needs a real fix (bunup investigation or per-subpath entry split),
  not a one-liner. **[INVESTIGATED — reverted, see §5]**
- **dice-ui web/native duplication** — extract `.util.ts` shared logic + a single token palette.

### CLARIFY (confusing / underspecified)

- **dice-ui dual-target true-north** — formalize dual-target (unify props + barrel-parity test)
  OR drop native for use-dom; update the stale `dice-ui/CLAUDE.md`.
- **`check:all` ≠ CI** — CLAUDE.md calls it "the full CI pipeline" and lists "size", but it runs
  neither knip nor size. Reconcile.
- **Doc drift** — README "Interactive TUI mode" (removed), llms.txt RDN v1.0 (only v0.9.0 exists),
  CONTRIBUTING `display-utils` (deleted), README TS badge 5.9 (actual 6.0.x). **[APPLIED — §5]**
- **Game spec input descriptions** — 5 of 6 specs omit `description`, degrading the _generated_
  validation error messages (not just docs).

## 8. Update — shipped on this branch (beyond the deps/security/docs refresh)

All four P1 backlog items plus the dice-ui crash were implemented and verified
(`bun run check` green across all 8 packages):

- **Count-modifier collision (P1, correctness)** — `parseModifiers` now rejects any 2+ count-family
  modifiers (`#{}`/`S{}`/`F{}`); shared `findCountFamilyMatchIndices` keeps `roll()` and
  `isDiceNotation` in agreement; conformance vector 47 un-parked; regression tests added.
- **Multi-roll-per-spec (P1)** — specs may declare `rolls: {name: …}` (mutually exclusive with
  `roll`); schema `oneOf` + identifier-key constraint, `getRollDefinitions`, type, and codegen
  remote-URL scan updated. Codegen/runtime already iterated named rolls. Tests added.
- **Dual-consumer drift test (P1)** — seeds `Math.random` identically for the generated and
  runtime paths across 5 games and asserts identical dice/total/result/details. **It caught two
  real runtime-interpreter bugs**, both fixed: additive `modify` ops were dropped from the total
  (resolveTotal summed dice only), and multiple `add` ops overwrote instead of accumulating.
- **Expo CI honesty (P1)** — root cause of the "~42 hidden failures" was bun `mock.module`
  cross-file pollution (each file passes alone). Fixed via `bun test --isolate`; repaired 5 real
  failures (stale theme tokens, WebHeader React-mock harness); added `test`/`check` scripts; CI
  now runs the full 86-test suite instead of one file.
- **dice-ui `DocModal` native crash (P1)** — the native barrel omitted `DocModal` (implemented but
  unexported), so the Expo app crashed at runtime while typechecking clean via the ambient shim.
  Exported it; added a static barrel-parity test + `test`/`check` scripts; wired dice-ui into CI.
